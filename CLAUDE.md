# StretchWidget — CLAUDE.md

## 프로젝트 개요

**앱 명칭:** StretchWidget (가칭)  
**플랫폼:** Windows PC 데스크톱 애플리케이션  
**목적:** 직장인·장시간 PC 사용자를 위한 정기 스트레칭 유도 위젯

**현재 스택:** React 19 + Vite (브라우저 프로토타입 단계)  
**목표 스택:** Electron 또는 Tauri + HTML/CSS/JavaScript

---

## 핵심 기능

### A. 단계별 알림 시스템

| 단계              | 트리거                     | 형태                        |
| ----------------- | -------------------------- | --------------------------- |
| 1단계 (토스트)    | 설정 주기마다 (기본 1시간) | 화면 우측 상단, 작고 반투명 |
| 2단계 (중앙 팝업) | 1단계 발생 후 1분간 미반응 | 화면 상단 중앙, 명확한 팝업 |

**팝업 버튼 3개:**

- **스트레칭 완료** — 알림 종료 후 다음 타이머 가동, 일일 게이지 +1
- **5분 뒤 다시 알리기** — 팝업 닫고 5분 후 1단계 재발생
- **방해 금지 모드** — 선택지: 1시간 / 2시간 / 3시간 / 4시간 / 오늘은 끄기

### B. 스케줄러

- 기본 가동 시간: **평일(월~금) 09:00 ~ 18:00**
- 가동 시간 외에는 타이머 정지, 시스템 트레이에서 대기
- 설정 메뉴에서 시간 및 요일 수정 가능

### C. 단축키

팝업 활성화 중 마우스 없이 제어 가능해야 함.

| 키                   | 동작               |
| -------------------- | ------------------ |
| `Space` 또는 `Enter` | 스트레칭 완료      |
| `Esc`                | 5분 뒤 다시 알리기 |

### D. 스트레칭 콘텐츠 및 게이지

**부위별 로테이션:** 알림마다 아래 부위를 순차 또는 랜덤으로 노출

- 목/어깨
- 손목/손가락
- 허리/골반

콘텐츠 형태: 간단한 텍스트 가이드 + CSS 애니메이션/도형 움직임

**일일 게이지 (Progress Bar):**

- 하루 목표 횟수: 기본 8회
- 스트레칭 완료 시마다 게이지 증가
- 가동 종료(18:00) 시 마감 메시지 노출 (예: "오늘도 건강을 지켰습니다!")

### E. 그룹 현황 대시보드 및 데이터 정책

**참여 및 동기화 방식:**

- 회원가입 없이 **[6자리 방 코드]**와 **[본인 닉네임]** 입력만으로 그룹 생성 및 참여
- 실시간 푸시 알림은 제외하며, 유저가 **대시보드 화면을 열 때만** 최신 데이터로 새로고침(On-Demand)

**대시보드 UI 구성:**

- **팀 전체 달성률:** 오늘 팀원 전체의 목표 달성률을 시각적 게이지바로 표현
- **팀원별 현황 리스트:** 오늘 완료 횟수순으로 정렬하며 상위권 유저에게 메달 아이콘 표시
  **무응답 유저 자동 보정 시스템 (팀 달성률 최적화):**
- 연차·출장 등으로 자리를 비운 유저로 인해 팀 달성률이 왜곡되는 것을 방지하기 위한 자동 필터링 로직 적용

| 유저 상태 구분       | 판단 조건                             | 팀 달성률 반영 로직                            |
| :------------------- | :------------------------------------ | :--------------------------------------------- |
| **정상 참여**        | 당일 알림에 지속적으로 반응함         | 개인 목표치 100% 반영 (기본 8회)               |
| **반차 / 단기 부재** | 최근 연속 3시간(3회 알림) 이상 무응답 | 당일 개인 목표치를 **50%로 하향 조정** (4회)   |
| **연차 / 종일 부재** | 당일 가동 시간 내 응답 횟수 총 0회    | 팀 전체 목표 총원(분모)에서 **해당 유저 제외** |

- 대시보드 팀원 리스트에서 3시간 이상 무응답 유저 옆에는 '자리를 비움' 상태 표시 연출

**데이터 누적 및 통계 정책:**

- 스트레칭 완료 시마다 독립적인 로그 기록 적재
- 대시보드 내 필터를 통해 **'지난주 랭킹(월~일)'** 등의 기간별 통계 집계 기능 제공

**데이터 스키마 (Firebase/Supabase 권장):**

| 테이블/컬렉션명   | 필드 구성                                                                    | 비고        |
| :---------------- | :--------------------------------------------------------------------------- | :---------- |
| `groups`          | `group_code` (PK), `created_at`                                              | 방 정보     |
| `users`           | `user_id` (PK), `group_code` (FK), `nickname`                                | 사용자 정보 |
| `stretching_logs` | `log_id` (PK), `user_id` (FK), `group_code` (FK), `completed_at` (Timestamp) | 완료 로그   |

**데이터 보존 정책 (TTL):**

- 서버 저장 공간 효율화 및 관리를 위해 `completed_at` 기준 **생성된 지 30일이 지난 오래된 로그 데이터는 자동으로 영구 삭제** 처리
